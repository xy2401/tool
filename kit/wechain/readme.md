# 微信群接龙统计助手

一个移动优先的 Vue3 静态网页工具，用来统计微信群官方接龙结果，帮助群主快速知道谁已经接龙、谁还没有接龙，并生成更有人情味的提醒文案。

项目不依赖微信接口，不需要后端登录。用户只需要把微信群里的接龙文本复制到网页中，网页自动解析、维护人员列表、更新每日打卡状态，并把数据保存在浏览器本地。

## 使用场景

适合健身群、学习群、读书群、打卡群、训练营等需要长期接龙统计的微信群。

微信群官方接龙有收集功能，但缺少统计能力。尤其在每日提醒未打卡人员时，群主需要手动比对人员名单和接龙名单，非常麻烦。本工具的目标是把这件事变成：

1. 选择日期。
2. 粘贴微信群接龙文本。
3. 自动识别已接龙人员。
4. 自动更新打卡表。
5. 一键生成完成或未完成提醒文案。

## 核心原则

- 移动优先，主要面向手机浏览器使用。
- 纯静态网页，使用 Vue3 驱动。
- 数据保存在浏览器本地，下次打开仍然可用。
- 不连接微信，不读取聊天记录，不需要登录。
- 人员列表可以通过多次粘贴接龙自然维护。
- 接龙人名尽量原样保留，不擅自裁剪数字、括号、符号。
- 提醒文案要有温度，完成时鼓励，未完成时提醒但不羞辱。

## 接龙数据示例

```text
#接龙

1. 小羊
2. 风里跑
3. 阿宁今天练腿
```

上面的示例表示三个人：

- 小羊
- 风里跑
- 阿宁今天练腿

解析时只去掉前面的接龙序号，序号后面的内容整体作为人名。

## 页面结构

页面主体是一张人员与日期组成的打卡矩阵。

```text
人员      06-01  06-02  06-03  06-04  06-05  06-06  06-07
          周一    周二    周三    周四    周五    周六    周日
          ○      ✓      -      ◎      ○      ○      ○
小羊        ✓      -      ✓      ✓      -      -      ✓
风里跑      ✓      ✓      -      ✓      ✓      -      ✓
阿宁今天练腿  -      ✓      ✓      -      ✓      ✓      ✓
```

第一列为人员列表，右侧为日期列。第一列固定，日期区域可以横向滚动。

日期范围第一版限定为当前日期前后各一个月。后续可以扩展成无限滚动日历。

## 日期列设计

为了节约移动端横向空间，每个日期表头分三行：

```text
06-01
周一
○
```

第三行是该日期的通知统计状态，使用简洁符号代替文字。

```text
○ 默认
✓ 完成
- 未完成
◎ 全选
```

点击日期表头第三行时，状态循环：

```text
○ -> ✓ -> - -> ◎ -> ○
```

状态含义：

- `○`：该日期不参与通知统计。
- `✓`：统计该日期或日期范围内全部完成的人。
- `-`：统计该日期或日期范围内未完成的人。
- `◎`：同时统计完成和未完成。

点击日期数字区域时，打开接龙文本粘贴弹窗，用于更新该日期的完成数据。

## 单元格状态

每个人每天的状态只使用符号显示。

```text
✓ 完成
- 未完成
```

点击单元格时状态循环：

```text
✓ -> - -> ✓
```

完成状态可以使用绿色勾号或实心圆点，未完成使用浅色横线或淡色圆点。删除人员整行使用暗色背景或降低透明度。

## 接龙粘贴与解析

用户选择某一天后，粘贴微信群接龙文本。

解析规则：

- 忽略空行。
- 忽略 `#接龙` 等标题行。
- 识别常见序号格式，例如 `1. 张三`、`2、李四`、`3 王五`。
- 只移除前面的序号。
- 序号后面的文本整体作为人名。
- 不自动删除尾部数字。
- 不自动改写括号内容。
- 不自动合并相似名称。
- 人员去重按完整显示名匹配。

如果解析到当前人员列表里不存在的新名字，页面提示是否加入当前列表。

## 人员列表维护

人员列表可以通过两种方式维护：

1. 多次粘贴接龙文本，自动发现新人。
2. 点击人员列表表头的新增入口，手动粘贴人员。

手动新增时，可以直接粘贴微信群里的 `@` 人文本。

```text
@小羊 @风里跑 @阿宁今天练腿
```

解析时保留 `@` 后面的显示名，不额外处理括号。

```text
小羊
风里跑
阿宁今天练腿
```

## 人员排序与分组

人员列表按字母和拼音自然排序，并显示分组标题。

```text
---a(10)---
阿明
andy

---b(20)---
Bob
冰冰

---z(3)---

---#(13)---

---删除(10)---
```

排序规则：

- 中文按拼音首字母归组。
- 英文按首字母归组。
- 数字、符号、无法识别的名称归到 `#`。
- 没有人员的字母分组不显示。
- 已删除人员放到 `删除` 分组。
- 分组标题显示该组人数。

## 删除与恢复

人员支持滑动删除。

第一次删除：

- 人员进入 `删除` 分组。
- 行背景变暗。
- 不参与完成/未完成统计。
- 历史打卡数据仍保留。

删除分组中的人员再次滑动删除：

- 从当前列表中彻底移除。
- 对应历史数据也可以清理。

删除分组中的人员应支持恢复到正常人员列表。

## 多人员列表

支持管理多份人员列表，例如：

- 健身群
- 学习群
- 读书群

顶部提供当前列表切换入口。

功能包括：

- 新建列表。
- 重命名列表。
- 删除列表。
- 复制当前列表。
- 切换列表。

每个列表独立保存：

- 列表名称。
- 正常人员。
- 删除人员。
- 日期打卡数据。
- 通知文案偏好。

## 日期选择与统计

多个日期可以同时参与统计。

当多个日期设置为 `✓` 时，统计这些日期全部完成的人。

当多个日期设置为 `-` 时，统计这些日期中存在未完成的人。

当日期设置为 `◎` 时，同时统计完成和未完成。

单日和多日生成不同文案。七天日期范围可以触发一周全勤文案。

## 通知文案生成

通知区域根据日期状态、统计结果、时间段和模板随机生成文案。

基础操作：

- 复制通知。
- 换一句。
- 复制 @ 名单。

模板变量：

```text
{date}        06-20
{range}       06-01 ~ 06-20
{days}        20
{count}       3
{names}       @小羊 @风里跑 @阿宁
{doneCount}   8
{missCount}   2
{doneNames}   @小羊 @风里跑
{missNames}   @阿宁 @大橙子
{timeSlot}    早上 / 中午 / 下午 / 晚上 / 深夜
{greeting}    早上好 / 午间提醒 / 下午加油 / 晚上收尾 / 夜深了
```

## 时间段判断

生成通知时，根据当前本地时间判断时间段。

```text
05:00 - 10:59 早上
11:00 - 13:59 中午
14:00 - 17:59 下午
18:00 - 22:59 晚上
23:00 - 04:59 深夜
```

通知区域可以提供语气时间选择：

```text
自动
早上
中午
下午
晚上
深夜
```

默认使用自动模式。用户也可以手动指定时间段。

## 文案语气

文案要有更多人文关怀，避免机械点名。

默认语气：

- 完成时鼓励。
- 未完成时提醒。
- 少一点批评，多一点陪伴感。
- 深夜提醒降低压力，注意休息。
- 一周或多日完成强调坚持被看见。

可以内置三种语气：

```text
温柔提醒
轻度鞭策
热血鼓励
```

第一版默认使用温柔提醒。

## 文案场景

建议内置以下模板池：

- 单日完成。
- 单日未完成。
- 多日完成。
- 多日未完成。
- 一周全勤。
- 全选统计。
- 空名单。

每类准备多条模板，生成时随机选取。点击“换一句”时只更换文案，不改变统计结果。

## 文案示例

单日完成：

```text
{date} 今日完成名单

{names}
{count}人已经完成接龙。坚持被看见了，今天这一卡很稳 🎉
```

单日未完成，早上：

```text
早上好，今天可以慢慢进入节奏。

{names}
还有{count}人未完成接龙，不急，但别忘了给自己一个开始。
```

单日未完成，中午：

```text
午间小提醒。

{names}
还有{count}人未完成接龙，吃完饭顺手补一下也刚好。
```

单日未完成，晚上：

```text
晚上收尾提醒。

{names}
还有{count}人未完成接龙，今天还没结束，补上就算数。
```

单日未完成，深夜：

```text
夜深了，轻轻提醒一下。

{names}
还有{count}人还没接龙。能补就补，太晚了也记得休息。
```

多日完成：

```text
{range} 共{days}天

{names}
{count}人完成全部接龙，连续稳定输出 🎉
```

多日未完成：

```text
{range} 打卡提醒

{names}
{count}人存在未完成记录。别急着追求完美，先把能补的那一格补上。
```

一周全勤：

```text
{range} 一周打卡

{names}
{count}人完成了一周全部接龙。不是每天都容易，但你们都做到了 🎉
```

全选统计：

```text
{range} 打卡统计

已完成全部接龙：{doneCount}人
{doneNames}

未完成全部接龙：{missCount}人
{missNames}

完成的继续稳住，没补齐的今天安排上。
```

空名单，没有未完成：

```text
{date}

今天没有未完成名单，全员到齐 🎉
```

空名单，没有完成人：

```text
{date}

暂时还没有人完成接龙，第一位从容开局的人在哪里。
```

## 鼓励动画

生成通知时，可以在通知区域加入轻量动画：

- 完成人名标签轻微浮起。
- 彩带或星星短暂出现。
- 一周全勤时出现更明显的庆祝动画。
- 未完成提醒时使用轻柔的提示动效，不做羞辱式动画。

动画只作为反馈，不影响复制文本。

## CSV 查看与导出

支持查看、复制和下载 CSV。

CSV 格式：

```csv
name,2026-06-01,2026-06-02,2026-06-03
小羊,1,1,0
风里跑,0,1,1
阿宁今天练腿,1,0,1
```

规则：

- 第一列是人名。
- 第二列开始是日期。
- 完成记为 `1`。
- 未完成记为 `0`。
- 删除人员可以选择不导出，或在高级选项中包含。

后续可以支持 CSV 导入，用于换浏览器或换设备迁移。

## 本地存储

第一版可以使用 `localStorage` 保存数据。

建议数据结构包含：

- 应用版本。
- 当前列表 ID。
- 所有人员列表。
- 每个列表的人员数据。
- 每个列表的打卡矩阵。
- 模板偏好与时间段偏好。

如果后续数据量变大，可以迁移到 `IndexedDB`。

## MVP 功能顺序

1. 移动优先页面布局。
2. 人员固定列和日期横向滚动矩阵。
3. 日期范围为当前日期前后各一个月。
4. 点击单元格切换完成/未完成。
5. 点击日期打开接龙粘贴弹窗。
6. 解析接龙文本并更新当天完成状态。
7. 发现新人时提示加入人员列表。
8. 手动新增人员，支持粘贴 `@` 人文本。
9. 人员按拼音和字母分组排序。
10. 支持删除、恢复、彻底删除人员。
11. 支持多人员列表切换和管理。
12. 日期表头状态支持 `○ / ✓ / - / ◎`。
13. 根据日期状态生成通知文案。
14. 支持按时间段随机文案。
15. 支持复制通知和换一句。
16. 支持 CSV 查看、复制和导出。
